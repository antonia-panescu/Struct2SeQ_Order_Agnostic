## Headline benchmark — architectural fix alone recovers the regression

Ran `evaluation/run_eval.py` on the new architecture, **loading
Struct2SeQ.pt's 209 pretrained weights but with the new RPE bias tables at
random init** (no retraining at all). 200 targets, len ≤ 200, 96 samples
per target via the standard 3-strategy protocol.

| Setup | MCC best-of-96 | Jaccard best-of-96 | sample_T1 best-of-32 |
|---|---|---|---|
| Untouched Struct2SeQ.pt + L→R inference (gold) | 0.935 | 0.920 | — |
| Untouched + old-LSTM-PE + randperm | 0.851 | 0.833 | — |
| Ep4 RL'd bidir + old-LSTM-PE + randperm (full 5407) | 0.832 | 0.812 | 0.677 |
| **Struct2SeQ.pt + NEW RPE arch + randperm (RPE *uninit*)** | **0.932** | **0.919** | **0.835** |

Per-strategy on the new arch (200-target subset):
- eps_argmax_p05: mcc_best_mean = 0.922, mcc_mean = 0.779
- eps_argmax_p10: mcc_best_mean = 0.918, mcc_mean = 0.770
- sample_T1:     mcc_best_mean = 0.835, mcc_mean = 0.538

This is the cleanest possible confirmation that the LSTM PE + CausalConv
mismatch was the dominant cause of the regression. Same encoder, same
attention QKV projections, same FFN, same fc_out — only the decoder's
position-encoding mechanism has changed (LSTM/Conv → relpos bias) — and
the model decodes in *random* RNA-position order at the same MCC as the
original L→R protocol. With the RPE bias tables completely untrained.

This also implies:
1. **RL retraining is NOT a prerequisite for fixing the regression.** The
   architectural fix is sufficient on its own. Whatever RL training adds
   will be on top of the 0.93 baseline, not closing a gap from 0.83.
2. **The model has all the information it needs in the encoder + cross
   attention to know which RNA position it's predicting**, as long as the
   decoder's position scheme doesn't actively mislead it (which the LSTM
   PE was doing under permutation).
3. **The catastrophic-forgetting hypothesis from yesterday is partially
   refuted**: the 10-pt gap that I attributed to forgetting was largely
   the architectural mismatch flowing through the RL'd checkpoint. With
   the new arch loaded from `Struct2SeQ.pt`, that gap collapses.

## Architectural details — RPE design

`RelativePositionBias` (T5-style), in `Encoder_Decoder.py:196`:
- 64 buckets, 12 heads, max_distance=512 — covers test set's L_max=493
- Bidirectional (32 buckets per sign) with 16 exact + 16 log-spaced per side
- Bias added to attention logits before softmax via
  `nn.MultiheadAttention(..., attn_mask=bias)` — additive logit bias
- Two separate tables: `rpe_self` (decoder ↔ decoder) and `rpe_cross`
  (decoder ↔ encoder); same architecture, separate parameters
- Param cost: 64 × 12 × 2 = 1,536 floats per bias table = ~12 KB total
- Initialized with `nn.init.normal_(std=0.02)` — small, lets the model
  learn what relpos signal is helpful

Self-attention bias indexing (the "PE[random_order, random_order]" trick):
```python
self_q_pos = perm[:, :L]                   # [B, L]
self_k_pos = perm[:, :L]                   # [B, L]
self_bias = self.rpe_self(self_q_pos, self_k_pos)   # [B, n_heads, L, L]
# rel_pos[b, i, j] = perm[b, j] - perm[b, i]   (RNA-position offset)
```

Cross-attention bias indexing (encoder is at fixed RNA positions):
```python
encoder_perm = arange(L_enc)                # [B, L_enc]
cross_bias = self.rpe_cross(perm[:, :L], encoder_perm)
# rel_pos[b, t, k] = k - perm[b, t]
```

KV-cache compatibility: each cached K/V was produced by an embedding that
does NOT depend on position (all position info is in the attention bias).
Cached K/V remains valid forever; at step t in the cached path, the
single new query at perm[t] gets bias row `rpe_self(perm[t], perm[:t+1])`.

## Code changes (committed in `822ebda`)

**`Encoder_Decoder.py`**
- New class `RelativePositionBias(num_buckets=64, max_distance=512, n_heads=12)`
- `OptimizedTransformerDecoder.__init__`: dropped `nn.LSTM` PE, dropped
  `CausalConv1d` + `LayerNorm`. Added `rpe_self` and `rpe_cross`.
- `OptimizedTransformerDecoder.forward(...)`: new args `perm`, `encoder_perm`.
  Computes self/cross biases and combines self-bias with the upper-triangular
  causal mask before passing to layers.
- `OptimizedDecoderLayer.forward(...)`: new signature
  `(tgt, memory, self_attn_mask, cross_attn_mask, ...)` — both passed through
  to `nn.MultiheadAttention(..., attn_mask=...)`. `need_weights=False`
  routes through PyTorch's SDPA backend.
- `DotBracketRNATransformer.forward(...)`: new `perm`, `encoder_perm` args
  threaded into the decoder; default to identity perm = L→R for backward
  compat.

**`Functions.py`**
- New canonical `generate_permuted` with corrected forward-looking
  `paired_encoding` convention (matches training's `paired_encoding_perm`,
  removes the 1-step lag confound that was in the eval pipeline).
- `generate_sequence_batched`, `_accelerate`, `_sample`: pass identity
  perm to the decoder so they still work as L→R generators.
- bf16-safe softmax-with-tiny-temperature: `mv32 = masked_values.float()`
  before `/1e7` to avoid bf16 underflow under autocast.

**`run.py`**
- `setup_models` / checkpoint-loading branch: `load_state_dict(strict=False)`
  after dropping the 8 legacy decoder-PE keys (LSTM weights, conv,
  conv_norm). Only RPE keys allowed in `missing`.
- `train_epoch` order-agnostic block: K-perm tiling
  (`config.k_perm`, default 1, set to 4 for production runs);
  per-sample `[B, L]` perms via Python loop of `torch.randperm`;
  `perm` threaded into both policy and target network forwards;
  paired_encoding convention unchanged (already forward-looking).
- `play` loop: replaced `generate_func` (the L→R generator) with
  `generate_permuted` from Functions.py; per-batch random perms.
- Startup: `torch.backends.cuda.sdp_kernel(enable_flash=True,
  enable_mem_efficient=True, enable_math=False)` to prefer flash/memeff
  attention.
- After `accelerator.prepare`: `torch.compile(policy_network, mode='default',
  dynamic=True)` (and same for target). Gated by env flag
  `STRUCT2SEQ_DISABLE_COMPILE=1` for emergency disable.
- Training DataLoader: num_workers 4→16, pin_memory=True,
  persistent_workers=True, prefetch_factor 2→4.

**`config_brev_8gpu.yaml`**
- `train_batch_size: 256 → 128`, `max_train_batch_size: 256 → 128`
- New: `k_perm: 4`

**`evaluation/run_eval.py`**
- Local `generate_permuted` is now a thin wrapper around the canonical
  `Functions.generate_permuted` (same forward-looking paired convention).
- Checkpoint loading: same DROP pattern as `run.py:setup_models`.
- Decode + folder wrapped in `torch.amp.autocast(bf16)`.
- bf16→fp32 cast before `.numpy()` to avoid the `BFloat16 ScalarType` error.

**New test script**: `evaluation/smoke_new_arch.py` — load Struct2SeQ.pt
into the new arch, verify DROP/missing keys, run L→R + randperm fwd+bwd,
exercise KV-cache for two steps, time 10 fwd+bwd at B=64 L=240
(reported 244 samp/s at smoke time).
